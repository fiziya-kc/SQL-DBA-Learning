-- Full backup (complete database)
BACKUP DATABASE SalesDB
TO DISK = 'D:\Backups\SalesDB_Full_20240115.bak'
WITH COMPRESSION, CHECKSUM, NAME = 'SalesDB Full Backup';

-- Differential backup (changes since last full)
BACKUP DATABASE SalesDB
TO DISK = 'D:\Backups\SalesDB_Diff_20240115.bak'
WITH DIFFERENTIAL, COMPRESSION, CHECKSUM;

-- Transaction log backup (requires FULL recovery model)
BACKUP LOG SalesDB
TO DISK = 'D:\Backups\SalesDB_Log_20240115_0900.trn'
WITH COMPRESSION;

Recovery Models

SQL Server Recovery Models

  • FULL - All transactions logged. Supports point-in-time recovery. Required for log backups.
  • BULK_LOGGED - Minimal logging for bulk operations. Some point-in-time limitations.
  • SIMPLE - No transaction log backup. Can only restore to last full/differential backup.
-- Check current recovery model
SELECT name, recovery_model_desc
FROM sys.databases
WHERE name = 'SalesDB';

-- Change recovery model to FULL
ALTER DATABASE SalesDB
SET RECOVERY FULL;

Restore Operations

-- Restore full backup
RESTORE DATABASE SalesDB
FROM DISK = 'D:\Backups\SalesDB_Full_20240115.bak'
WITH RECOVERY, REPLACE;

-- Restore full + differential
RESTORE DATABASE SalesDB
FROM DISK = 'D:\Backups\SalesDB_Full_20240115.bak'
WITH NORECOVERY;

RESTORE DATABASE SalesDB
FROM DISK = 'D:\Backups\SalesDB_Diff_20240115.bak'
WITH RECOVERY;

-- Restore with transaction log (point-in-time)
RESTORE DATABASE SalesDB
FROM DISK = 'D:\Backups\SalesDB_Full_20240115.bak'
WITH NORECOVERY;

RESTORE LOG SalesDB
FROM DISK = 'D:\Backups\SalesDB_Log_20240115.trn'
WITH RECOVERY,
     STOPAT = '2024-01-15 09:30:00';

-- Verify restore
RESTORE VERIFYONLY
FROM DISK = 'D:\Backups\SalesDB_Full_20240115.bak';

Critical: Always use WITH CHECKSUM when creating backups. It validates page checksums and detects corruption early. Test your backups regularly by actually restoring them!

Backup Strategy Example

Common Backup Schedule (FULL Recovery)

  • Daily Full Backup - Every day at 2:00 AM
  • Hourly Log Backups - Every hour during business hours
  • Weekly Verification - Test restore on development server
  • Offsite Copy - Copy backups to cloud storage daily

Best Practice: Follow the 3-2-1 backup rule: 3 copies of data, on 2 different media types, with 1 copy offsite.
