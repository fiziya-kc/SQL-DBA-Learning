What is a Trigger?

A trigger is a special stored procedure that automatically executes when a specific event occurs in the database, such as INSERT, UPDATE, or DELETE operations.

Types of Triggers

  • DML Triggers - Fire on INSERT, UPDATE, DELETE
  • DDL Triggers - Fire on CREATE, ALTER, DROP
  • AFTER - Fires after the action completes
  • INSTEAD OF - Fires instead of the action

DML Triggers

-- Audit trigger for tracking changes
CREATE TRIGGER trg_EmployeeAudit
ON employees
AFTER INSERT, UPDATE
AS
BEGIN
    INSERT INTO employee_audit (
        employee_id, action, old_salary, new_salary, 
        changed_by, changed_date
    )
    SELECT 
        i.employee_id,
        CASE WHEN EXISTS(SELECT 1 FROM deleted) 
            THEN 'UPDATE' ELSE 'INSERT' END,
        d.salary,
        i.salary,
        SYSTEM_USER,
        GETDATE()
    FROM inserted i
    LEFT JOIN deleted d ON i.employee_id = d.employee_id;
END

-- INSTEAD OF trigger (for views or constrained inserts)
CREATE TRIGGER trg_PreventDelete
ON employees
INSTEAD OF DELETE
AS
BEGIN
    RAISERROR('Direct deletes are not allowed. Use soft delete.', 16, 1);
END

Caution: Use triggers sparingly. They can cause performance issues and make debugging difficult. Business logic in application code is usually easier to maintain.
