The foundation of all SQL queries. Learn how to retrieve, filter, and transform data from database tables.

What is SELECT?

The SELECT statement is the most fundamental and frequently used SQL command. It allows you to query and retrieve data from one or more database tables. Every data analysis, report, and data-driven application starts with SELECT.

Basic Syntax

SELECT column1, column2, column3
FROM table_name;

-- Select ALL columns
SELECT * FROM table_name;

Practical Examples

-- Example 1: Select all columns from employees table
SELECT * FROM employees;

-- Example 2: Select specific columns
SELECT first_name, last_name, email, hire_date
FROM employees;

-- Example 3: Use column aliases with AS
SELECT 
    first_name AS 'First Name',
    last_name AS 'Last Name',
    salary * 12 AS annual_salary,
    department_id AS dept_no
FROM employees;

-- Example 4: Concatenate columns
SELECT 
    first_name + ' ' + last_name AS full_name
FROM employees;

-- Example 5: Use expressions in SELECT
SELECT 
    product_name,
    unit_price,
    quantity,
    unit_price * quantity AS total_value
FROM order_items;

Filtering with WHERE

The WHERE clause filters rows based on specified conditions. Only rows that meet the criteria are included in the result set.

-- Filter with single condition
SELECT first_name, last_name, salary
FROM employees
WHERE salary > 50000;

-- Filter with text value (use single quotes)
SELECT *
FROM employees
WHERE department = 'Sales';

-- Multiple conditions with AND
SELECT *
FROM employees
WHERE department = 'Sales'
   AND salary > 60000
   AND hire_date > '2020-01-01';

-- Multiple conditions with OR
SELECT *
FROM employees
WHERE department = 'Sales'
   OR department = 'Marketing';

-- Combine AND and OR (use parentheses!)
SELECT *
FROM employees
WHERE (salary > 80000 OR department = 'HR')
   AND is_active = 1;

-- NOT operator
SELECT *
FROM employees
WHERE NOT department = 'IT';

Pro Tip: Always use parentheses when combining AND and OR to ensure correct evaluation order. AND has higher precedence than OR.

Comparison Operators

Available Operators

  • = (Equal) - salary = 50000
  • <> or != (Not equal) - department <> 'Sales'
  • > (Greater than) - salary > 50000
  • < (Less than) - salary < 80000
  • >= (Greater than or equal) - salary >= 50000
  • <= (Less than or equal) - salary <= 80000
  • BETWEEN (Range) - salary BETWEEN 50000 AND 80000
  • IN (Multiple values) - department IN ('Sales', 'Marketing')
  • LIKE (Pattern match) - name LIKE 'J%'
  • IS NULL / IS NOT NULL (Null check) - email IS NOT NULL
-- BETWEEN operator (inclusive)
SELECT *
FROM employees
WHERE salary BETWEEN 50000 AND 80000;

-- IN operator (matches any in list)
SELECT *
FROM employees
WHERE department IN ('Sales', 'Marketing', 'HR');

-- LIKE operator for pattern matching
-- % matches any sequence of characters
SELECT *
FROM employees
WHERE last_name LIKE 'Sm%';      -- Starts with Sm

SELECT *
FROM employees
WHERE email LIKE '%@gmail.com'; -- Ends with @gmail.com

-- _ matches exactly one character
SELECT *
FROM employees
WHERE first_name LIKE 'J_hn';    -- Matches "John"

-- NULL handling
SELECT *
FROM employees
WHERE manager_id IS NULL;    -- No manager (top level)

SELECT *
FROM employees
WHERE phone IS NOT NULL;    -- Has a phone number

Sorting with ORDER BY

-- Sort ascending (default)
SELECT first_name, last_name, salary
FROM employees
ORDER BY salary;

-- Sort descending
SELECT first_name, last_name, hire_date
FROM employees
ORDER BY hire_date DESC;

-- Sort by multiple columns
SELECT department, first_name, last_name, salary
FROM employees
ORDER BY department ASC, salary DESC;

-- Sort by column position (not recommended for clarity)
SELECT first_name, last_name, salary
FROM employees
ORDER BY 3 DESC;  -- Sorts by 3rd column (salary)

Limiting Results

-- SQL Server: TOP keyword
SELECT TOP 10 *
FROM employees
ORDER BY salary DESC;

-- SQL Server: TOP with percent
SELECT TOP 10 PERCENT *
FROM employees
ORDER BY salary DESC;

-- SQL Server: WITH TIES (includes ties)
SELECT TOP 10 WITH TIES *
FROM employees
ORDER BY salary DESC;

-- MySQL/PostgreSQL: LIMIT
SELECT *
FROM employees
ORDER BY salary DESC
LIMIT 10 OFFSET 0;

-- Pagination example
SELECT *
FROM employees
ORDER BY employee_id
LIMIT 10 OFFSET 20;  -- Page 3 (skip first 20, get next 10)

DISTINCT - Removing Duplicates

-- Get unique department values
SELECT DISTINCT department
FROM employees;

-- Get unique combinations
SELECT DISTINCT department, job_title
FROM employees;

-- Count unique values
SELECT COUNT(DISTINCT department) AS num_departments
FROM employees;

Performance Warning: SELECT * retrieves all columns and should be avoided in production code. It increases network traffic, memory usage, and makes your code less maintainable. Always explicitly list the columns you need.

Key Points to Remember

  • SELECT retrieves data, it never modifies data
  • WHERE filters rows before they're returned (reduces data early)
  • ORDER BY sorts results (happens after filtering)
  • String values must be enclosed in single quotes
  • SQL is case-insensitive for keywords, but case-sensitive for data values
  • DISTINCT removes duplicate rows from results
  • LIMIT/TOP restricts the number of rows returned

Execution Order: SQL executes in this order: FROM → WHERE → SELECT → ORDER BY. Understanding this helps you write efficient queries and debug issues.
