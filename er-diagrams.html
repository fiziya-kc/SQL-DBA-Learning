What is an ER Diagram?

ER (Entity-Relationship) diagrams are visual tools used to design and represent database schemas. They show entities (tables), their attributes (columns), and the relationships between them.

ER Diagram Components

Rectangle

Entity (Table)

Ellipse

Attribute (Column)

Diamond

Relationship

Underline

Primary Key

Dashed Line

Non-identifying

Solid Line

Identifying

Relationship Cardinality

1:1 (One-to-One)

Each record relates to exactly one record

1:N (One-to-Many)

One record relates to many records

N:N (Many-to-Many)

Many records relate to many records

Real-World Example: E-Commerce Database

Tools: Use dbdiagram.io, Lucidchart, Draw.io, or SQL Server's Database Diagram feature to create professional ER diagrams. These tools can often generate SQL from your diagrams!

Best Practices

ER Diagram Design Guidelines

  • Identify all entities (nouns) in your system - these become tables
  • Identify relationships (verbs) between entities
  • Determine cardinality (1:1, 1:N, N:N) for each relationship
  • Identify attributes for each entity - these become columns
  • Mark primary keys clearly
  • Document business rules in notes
  • Use consistent naming conventions
