Master aggregate functions and grouping to summarize, analyze, and derive insights from large datasets.

Aggregate Functions

Aggregate functions perform calculations on sets of rows and return a single result. They're essential for reporting, analytics, and data analysis.

COUNT()

Counts rows or non-null values

SUM()

Adds all numeric values

AVG()

Calculates the average

MIN()

Finds the minimum value

MAX()

Finds the maximum value

COUNT_BIG()

For large result sets

-- Basic aggregate functions
SELECT
    COUNT(*) AS total_rows,
    COUNT(DISTINCT department) AS num_departments,
    SUM(salary) AS total_payroll,
    AVG(salary) AS average_salary,
    MIN(salary) AS lowest_salary,
    MAX(salary) AS highest_salary
FROM employees;

-- COUNT variations
SELECT
    COUNT(*) AS all_rows,              -- Includes NULLs
    COUNT(manager_id) AS with_manager,     -- Excludes NULLs
    COUNT(DISTINCT department) AS unique_depts
FROM employees;

GROUP BY Clause

GROUP BY groups rows with the same values into summary rows. It's used with aggregate functions to perform calculations per group.

-- Group by single column
SELECT
    department,
    COUNT(*) AS employee_count,
    AVG(salary) AS avg_salary,
    SUM(salary) AS total_salary
FROM employees
GROUP BY department
ORDER BY avg_salary DESC;

-- Group by multiple columns
SELECT
    department,
    job_title,
    COUNT(*) AS headcount,
    AVG(salary) AS avg_salary
FROM employees
GROUP BY department, job_title
ORDER BY department, headcount DESC;

-- GROUP BY with WHERE (filter before grouping)
SELECT
    department,
    COUNT(*) AS emp_count
FROM employees
WHERE hire_date >= '2022-01-01'
GROUP BY department;

HAVING vs WHERE

Understanding the Difference

  • WHERE - Filters individual rows before grouping
  • HAVING - Filters grouped results after aggregation
  • You can use both together: WHERE first (rows), then HAVING (groups)
-- WHERE filters rows BEFORE grouping
SELECT
    department,
    AVG(salary) AS avg_salary
FROM employees
WHERE hire_date > '2020-01-01'   -- Filter rows first
GROUP BY department;

-- HAVING filters groups AFTER aggregation
SELECT
    department,
    AVG(salary) AS avg_salary
FROM employees
GROUP BY department
HAVING AVG(salary) > 70000;    -- Filter groups second

-- Using both WHERE and HAVING together
SELECT
    department,
    COUNT(*) AS emp_count,
    AVG(salary) AS avg_salary
FROM employees
WHERE is_active = 1                    -- 1. Filter rows
GROUP BY department                      -- 2. Group them
HAVING COUNT(*) >= 3                  -- 3. Filter groups
ORDER BY avg_salary DESC;             -- 4. Sort results

Advanced Aggregate Functions

-- STRING_AGG - Concatenate values within group
SELECT
    department,
    STRING_AGG(first_name, ', ') WITHIN GROUP (ORDER BY first_name) AS employees
FROM employees
GROUP BY department;

-- GROUPING SETS - Multiple grouping levels
SELECT
    department,
    job_title,
    SUM(salary) AS total_salary
FROM employees
GROUP BY GROUPING SETS (
    (department),              -- By department
    (job_title),              -- By job title
    (department, job_title),   -- Both
    ()                         -- Grand total
);

-- ROLLUP - Hierarchical aggregation
SELECT
    region,
    department,
    SUM(sales) AS total_sales
FROM sales_data
GROUP BY ROLLUP (region, department);

-- CUBE - All combinations
SELECT
    year,
    quarter,
    SUM(revenue) AS total_revenue
FROM quarterly_sales
GROUP BY CUBE (year, quarter);

Window Functions (OVER)

Window functions perform calculations across a set of rows related to the current row, without reducing the number of rows returned.

-- Calculate running total
SELECT
    order_date,
    amount,
    SUM(amount) OVER (ORDER BY order_date) AS running_total
FROM orders;

-- Calculate rank within group
SELECT
    first_name,
    department,
    salary,
    RANK() OVER (PARTITION BY department ORDER BY salary DESC) AS salary_rank,
    DENSE_RANK() OVER (PARTITION BY department ORDER BY salary DESC) AS dense_rank
FROM employees;

-- Compare to average
SELECT
    first_name,
    salary,
    department,
    AVG(salary) OVER (PARTITION BY department) AS dept_avg,
    salary - AVG(salary) OVER (PARTITION BY department) AS diff_from_avg
FROM employees;

-- ROW_NUMBER with different orderings
SELECT
    first_name,
    hire_date,
    ROW_NUMBER() OVER (ORDER BY hire_date) AS seniority,
    ROW_NUMBER() OVER (PARTITION BY department ORDER BY salary DESC) AS top_paid_in_dept
FROM employees;

Pro Tip: Always include all non-aggregated columns in your GROUP BY clause. In SQL Server, this is strictly enforced. Other databases may return unpredictable values for columns not in GROUP BY.

Execution Order: FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY. Understanding this order helps you debug queries and write efficient aggregation.
