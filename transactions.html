What is a Transaction?

A transaction is a group of operations performed as a single unit of work. All operations must succeed for the transaction to commit, or all fail if any operation fails (rollback).

ACID Properties

  • Atomicity - All or nothing - transactions are indivisible
  • Consistency - Database moves from one valid state to another
  • Isolation - Concurrent transactions don't interfere
  • Durability - Committed changes survive system failures

Transaction Syntax

-- Basic transaction
BEGIN TRANSACTION;

UPDATE accounts SET balance = balance - 100 WHERE account_id = 1;
UPDATE accounts SET balance = balance + 100 WHERE account_id = 2;

COMMIT;

-- Transaction with error handling
BEGIN TRY
    BEGIN TRANSACTION;
    
    INSERT INTO orders (customer_id, total) VALUES (100, 500);
    UPDATE inventory SET quantity = quantity - 1 WHERE product_id = 50;
    
    COMMIT;
END TRY
BEGIN CATCH
    ROLLBACK;
    SELECT ERROR_MESSAGE() AS ErrorMessage;
END CATCH

Tip: Keep transactions short. Long-running transactions can cause blocking and lock contention. Always handle errors with TRY/CATCH and rollback on failure.
