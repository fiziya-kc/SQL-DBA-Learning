Index Optimization

Speed up queries dramatically with strategic index design and maintenance.

What is an Index?

An index is a data structure that improves the speed of data retrieval operations on a database table. Think of it like a book's index - it helps you find information faster without reading every page.

Clustered Index

Defines the physical order of data in a table. Only ONE per table. Best for primary keys and range queries.

Non-Clustered Index

Separate from data storage. Up to 999 per table. Best for WHERE clauses, JOINs, and ORDER BY.

Covering Index

Includes all columns needed by a query. Eliminates table lookups completely.

Filtered Index

Indexes only a subset of rows. Great for sparse columns with common values.

Creating Indexes

-- Non-clustered index (most common)
CREATE NONCLUSTERED INDEX IX_Employees_LastName
ON employees(last_name);

-- Composite index (multiple columns)
CREATE NONCLUSTERED INDEX IX_Orders_CustomerDate
ON orders(customer_id, order_date);

-- Covering index (includes additional columns)
CREATE NONCLUSTERED INDEX IX_Employees_Dept_Covering
ON employees(department_id)
INCLUDE (first_name, last_name, salary);

-- Filtered index (subset of rows)
CREATE NONCLUSTERED INDEX IX_Employees_Active
ON employees(last_name)
WHERE is_active = 1;

-- Unique index
CREATE UNIQUE INDEX UIX_Employees_Email
ON employees(email);

Finding Missing Indexes

-- Query DMF for missing indexes
SELECT
    migs.avg_user_impact * migs.avg_total_user_cost AS improvement_score,
    migs.avg_total_user_cost,
    migs.avg_user_impact,
    migs.user_seeks,
    migs.unique_compiles,
    mid.statement AS table_name,
    mid.equality_columns,
    mid.inequality_columns,
    mid.included_columns
FROM sys.dm_db_missing_index_groups mig
INNER JOIN sys.dm_db_missing_index_group_stats migs 
    ON mig.index_group_handle = migs.group_handle
INNER JOIN sys.dm_db_missing_index_details mid
    ON mig.index_handle = mid.index_handle
WHERE migs.user_seeks > 100
ORDER BY improvement_score DESC;

Analyzing Index Usage

-- Find unused indexes (SQL Server 2016+)
SELECT
    OBJECT_NAME(i.object_id) AS table_name,
    i.name AS index_name,
    i.type_desc,
    user_seeks,
    user_scans,
    user_lookups,
    user_updates,
    i.is_primary_key
FROM sys.indexes i
LEFT JOIN sys.dm_db_index_usage_stats us
    ON i.object_id = us.object_id
    AND i.index_id = us.index_id
    AND us.database_id = DB_ID()
WHERE OBJECTPROPERTY(i.object_id, 'IsUserTable') = 1
ORDER BY user_seeks + user_scans ASC;

Index Maintenance

-- Rebuild index (online in Enterprise edition)
ALTER INDEX IX_Employees_LastName
ON employees
REBUILD;

-- Reorganize index (lighter maintenance)
ALTER INDEX IX_Employees_LastName
ON employees
REORGANIZE;

-- Rebuild all indexes on a table
ALTER INDEX ALL ON employees
REBUILD;

-- Check index fragmentation
SELECT
    OBJECT_NAME(object_id) AS table_name,
    index_id,
    name AS index_name,
    avg_fragmentation_in_percent,
    page_count
FROM sys.dm_db_index_physical_stats(
    DB_ID(), NULL, NULL, NULL, 'DETAILED'
) dps
INNER JOIN sys.indexes i ON dps.object_id = i.object_id AND dps.index_id = i.index_id
WHERE avg_fragmentation_in_percent > 30
ORDER BY avg_fragmentation_in_percent DESC;

Caution: Indexes improve read performance but slow down INSERT, UPDATE, and DELETE operations. Each index adds overhead. Use the Missing Index DMVs and execution plans to guide your index creation.

Index Design Tips: Put the most selective column first. Include columns used in WHERE, JOIN, and ORDER BY. Use covering indexes for frequent queries. Remove duplicate/unused indexes.
